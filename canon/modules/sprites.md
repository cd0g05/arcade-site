# Module: src/sprites (sprite pipeline)

> Generated by Cicadas at arcade-buildout initiative close.

## Purpose

Build-time compiler turning small ASCII palette maps into committed inline-SVG modules, so no game generates SVG at runtime and every pixel-art icon in the product traces back to one source-of-truth file.

## Interfaces

- **Exports**: `maps.ts` (source ASCII maps, one per sprite, palette-char-only); `generated.ts` (committed output — `SPRITES: Record<SpriteName, string>` of inline SVG markup); `scripts/gen-sprites.ts` (the compiler, run via `npm run sprites`).
- **Dependencies**: none at runtime for consumers (`generated.ts` is plain strings); the generator itself runs via plain Node (type-stripping, no build step of its own).

## Key Decisions

- **ASCII-map source of truth, SVG is a build artifact**: adding a sprite means editing `maps.ts` and running `npm run sprites` — never hand-editing `generated.ts`.
- **8 sprites total**: `joystick` (hub logo), `token` (Miner), `snake`, `target` (Aim Trainer), `tube` (Water Sort), `bricks`, `invader`, `setrit` — decoded from the original mockup's inline SVGs where possible (tube/bricks/invader/joystick/token/snake/target), drawn fresh where the mockup had no icon (setrit).
- **Memory (the hub cartridge) uses all 8 sprites as its pair-matching faces** — the sprite sheet is functional game content, not just decoration.
- **Palette-only fills, `crispEdges` rendering, RLE-compressed rects**: specs guard staleness (generated output matches `maps.ts`), palette-only fill colors, and crisp-edges shape-rendering so pixel art never gets anti-aliased soft at any zoom.
