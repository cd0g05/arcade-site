# Module: src/games (the 12 games)

> Generated by Cicadas at arcade-buildout initiative close.

## Purpose

One directory per game, each independently authored against the shared `src/lib`/`src/ui` contracts. Six mount as hub cartridges (embedded cards on `/`); six are cabinet games (their own page under `/games/{slug}/`). Every game with meaningful rules separates them into a pure, unit-tested `logic.ts` from the DOM/canvas UI in `{game}.ts`.

## Interfaces

- **Exports**: each game exports either `mount{Game}(card: GameCard): MountedGame` (hub cartridges) or `def: CabinetDef` (cabinet games) — see `src/games/types.ts` for both shapes.
- **Dependencies**: `src/lib` (loop/screen/storage/audio/input/cartridge), `src/ui/card.ts` (the `GameCard` it mounts into), `src/sprites/generated.ts` (Memory uses all 8 sprites as pair faces; a few cards use one decoratively).

## Key Decisions

- **Pure `logic.ts` wherever rules are worth testing**: 2048, Miner economics, Lights Out, Snake, Minesweeper, Water Sort, Setrit all separate rules (no DOM, injectable `rand`) from rendering. Dino, Bricks, Aim, Echo, Memory skip this split — their "rules" are procedural/continuous physics, not discrete state machines worth a spec suite.
- **Game-owned CSS**: since `src/styles` is frozen, every game since foundation ships its own `{game}.css` imported by its module (`echo.css`, `memory.css`, `lightsout.css`, `minesweeper.css`, `watersort.css`, `setrit.css`).
- **Deterministic specs via injected LCG, not `Math.random`**: every `logic.ts` spec file seeds a small linear congruential generator so randomized behavior (mine placement, tile spawns, bag order, reverse-pour generation) is reproducible across runs.
- **Cabinet game-over = sleep + veil restyle, not a modal**: on death/win, the cartridge waits ~1s then calls the equivalent of `Hub.sleep()` itself and rewrites its veil copy to a result message — this is how nav keys get released back to the page without any special-case Hub logic.
- **Awake-gating on pointer input**: cabinet games gate `pointerdown` handlers on `Hub.current === id`; because the surface's own `pointerdown` fires before the Hub's wake `mousedown`, the waking click is a no-op for game input for free (no explicit "ignore first click" flag needed).
- **Water Sort's solvability guarantee**: levels are generated by reverse-pouring from a solved state with every reverse step constrained to be exactly invertible — `generate()` returns the forward solution alongside the level, and the test suite proves solvability by replaying it, not by hoping the generator is correct.
- **Setrit is intentionally not full-SRS**: a simple offset kick table (`[0,0]/[±1,0]/[0,-1]/[±2,0]`), timeboxed to "standard-feeling" per an explicit tech-design risk note rather than guideline-perfect rotation.
- **Naming doctrine applied per-game**: Setrit (not Tetris), Bricks (not Breakout), Echo (not Simon — live Hasbro trademark, verified via search mid-initiative and renamed with no save-migration needed since nothing was deployed yet).
