# Module: src/ui (shared chrome)

> Generated by Cicadas at arcade-buildout initiative close.

## Purpose

Shared DOM chrome so no individual game reinvents its own card, veil, stats row, or cabinet page frame. `card.ts` builds the `.gcard` structure used by every hub cartridge; `cabinet.ts` promotes that same visual vocabulary to full-page scale for cabinet games (header, `← HUB` link, screen, controls legend). `scoreboard.ts` and `ticker.ts` are the small hub-only live-data widgets. Frozen alongside `src/lib` — game-specific CSS lives with each game, never here.

## Interfaces

- **Exports**:
  - `card.ts` → `createCard(opts: CardOptions) → GameCard { root, body, foot, veil, setStat(label, value), flashStat(label), setVeil(msg, sub?) }`
  - `cabinet.ts` → `createCabinet(root, opts: CabinetOptions) → CabinetPage { card: GameCard, main }`
  - `scoreboard.ts` → `createScoreboard(rows: Record<string, () => string>) → { refresh() }`
  - `ticker.ts` → `fillTicker(items: string[])`
- **Dependencies**: `src/lib/audio.ts` (cabinet's sound toggle), no game-specific imports — this module knows nothing about individual games.

## Key Decisions

- **All dynamic text goes through `textContent`, never variable `innerHTML`** — the only `innerHTML` writes in this module are build-time-constant SVG strings (the ⛶ icon), so there is no DOM-injection surface even though the product has zero backend to worry about.
- **The Hub reads DOM structure this module builds** (`.veil`, `.veil-msg`, `.led`, `.fs-btn`, `[data-status]`) rather than the two modules sharing a typed contract — `hub.ts` tolerates missing pieces, so `card.ts` changes are additive-safe as long as those selectors stay put.
- **Cabinet pages are near-empty HTML shells** rendered entirely by `createCabinet()` at runtime (via `cabinet-entry.ts`) — chrome literally cannot drift between cabinet games because there's no per-page markup to drift.
- **`aria-live="polite"` status line** lives on every card (`[data-status]`); the Hub writes "Playing." / "Paused." / "Fullscreen." into it on every state transition — this is the sole a11y announcement mechanism, no separate live-region singleton.
